# tic-tac-toe-8051
Tic-Tac-Toe game on AT89C51 microcontroller using 8051 assembly with LCD display, matrix keypad, and AI opponent
# 🎮 Tic-Tac-Toe — AT89C51 Assembly ![Language](https://img.shields.io/badge/Language-8051%20Assembly-blue) ![MCU](https://img.shields.io/badge/MCU-AT89C51-gold) ![Simulator](https://img.shields.io/badge/Simulator-Proteus%20DSP-orange) ![IDE](https://img.shields.io/badge/IDE-Mide--51-green) ![License](https://img.shields.io/badge/License-MIT-lightgrey) A fully functional Tic-Tac-Toe game implemented in bare-metal 8051 assembly for the AT89C51 microcontroller. Features a 4×4 matrix keypad for input, LM041L LCD for display, two game modes (PvP and PvAI), and an intelligent AI opponent with a 5-level priority strategy. --- ## 📸 Demo ![Circuit Diagram](docs/circuit-diagram.png) ![LCD Output](docs/lcd-display.png) --- ## ✨ Features - ♟️ **Two Game Modes** — Player vs Player (PvP) and Player vs AI (PvAI) - 🤖 **AI Opponent** — 5-level priority strategy: Win → Block → Center → Corners → Edges - 🖥️ **LCD Display** — Real-time board rendering on LM041L 4-line LCD - ⌨️ **Matrix Keypad** — 3×3 cell selection via 4×4 keypad (keys 1–9) - 🔄 **Mode Toggle** — Single button (P3.7) switches mode and resets board - ⚡ **Bare Metal** — No OS, no libraries; pure 8051 assembly (1044 lines) - 🏆 **Win Detection** — All 8 lines checked (3 rows, 3 cols, 2 diagonals) - 🤝 **Draw Detection** — Detects board-full with no winner --- ## 🛠️ Hardware | Component | Details | |----------------|----------------------------------------------| | Microcontroller | AT89C51 (8051-compatible, 4KB Flash) | | Display | LM041L LCD — 4-line, 8-bit bus via P1 | | Input | 4×4 Matrix Keypad (3×3 cells used) | | Clock | Crystal oscillator X1 + 22pF caps (C1, C2) | | Mode Button | P3.7 — Active LOW, debounced | | Simulator | Proteus DSP | --- ## 📌 Pin Mapping | Signal | MCU Pin | Description | |----------------|------------|----------------------------------------| | LCD Data D0–D7 | P1.0–P1.7 | 8-bit data bus | | LCD RS | P2.0 | Register Select (0=Cmd, 1=Data) | | LCD Enable | P2.2 | Strobe to latch data | | LCD RW | GND | Write-only (tied low) | | Keypad Row 1 | P2.1 | Drive LOW → scan keys 1,2,3 | | Keypad Row 2 | P2.3 | Drive LOW → scan keys 4,5,6 | | Keypad Row 3 | P2.4 | Drive LOW → scan keys 7,8,9 | | Keypad Col 1–3 | P2.5–P2.7 | Column detect (pull-up required) | | Mode Button | P3.7 | Active LOW — toggle game mode | --- ## 🧠 Memory Architecture ### Board Storage The 8051 has 8 working registers (R0–R7). All 9 board cells are mapped as: | Register | Address | Cell | Position | |----------|---------|------|--------------| | R0 | 00h | 1 | Top-Left | | R1 | 01h | 2 | Top-Center | | R2 | 02h | 3 | Top-Right | | R3 | 03h | 4 | Mid-Left | | R4 | 04h | 5 | **CENTER ★** | | R5 | 05h | 6 | Mid-Right | | R6 | 06h | 7 | Bot-Left | | R7 | 07h | 8 | Bot-Center | | **DPL** | SFR 82h | 9 | Bot-Right | > **DPL Trick:** Since the board needs 9 slots but only 8 registers exist, > `DPL` (low byte of DPTR, SFR at 82h) is repurposed as Cell 9. > DPTR is unused since there is no external RAM in this project. ### Cell Values - `00h` = Empty - `58h` = 'X' (ASCII) - `4Fh` = 'O' (ASCII) ### Special Registers - `B` (SFR F0h) — Active player token (58h='X', 4Fh='O') - `MODEAI` — Bit flag at RAM bit-address 00h (byte 20h.0) — PvP vs PvAI --- ## 🤖 AI Strategy The AI uses a deterministic priority waterfall (no randomness): 1. **WIN** — Complete a line (2× 'O' + 1 empty → play there) 2. **BLOCK** — Stop the player (2× 'X' + 1 empty → block it) 3. **CENTER** — Take cell 5 (R4) if empty — intersects 4 win lines 4. **CORNERS** — Try R0 → R2 → R6 → DPL (each on 3 win lines) 5. **EDGES** — Take R1 → R3 → R5 → R7 as last resort --- ## 🗂️ Project Structure ``` tic-tac-toe-8051/ ├── src/ │ └── tic10_fixed.asm ← Main 8051 assembly source (1044 lines) ├── simulation/ │ └── tcoe.pdsprj ← Proteus DSP project file ├── hex/ │ └── tic10_fixed.hex ← Compiled Intel HEX output ├── docs/ │ ├── circuit-diagram.png ← Proteus schematic screenshot │ └── pin-mapping.md ← Port & pin reference └── README.md ``` --- ## 🚀 How to Run ### Simulation (Proteus) 1. Install **Proteus DSP** (version 8 or above) 2. Open `simulation/tcoe.pdsprj` 3. Click the **Play** button to start simulation 4. Use the on-screen keypad to play (keys 1–9 for cells) 5. Press **P3.7** button to toggle between PvP and PvAI modes ### Compile from Source 1. Install **Mide-51** (free 8051 assembly IDE) 2. Open `src/tic10_fixed.asm` 3. Press **F9** or click Build → Compile 4. The `.hex` file is generated in the output folder 5. Load the `.hex` into Proteus or flash to real hardware --- ## ⚠️ Known Constraints & Solutions | Challenge | Solution | |-----------|----------| | 9 cells but only 8 registers | DPL (SFR 82h) repurposed as Cell 9 | | ACALL address range errors | All ACALL → LCALL, all SJMP → LJMP (64KB reach) | | Keypad contact bounce | SMALLDEB (30-iter loop) + WAITREL until columns go HIGH | | Code bloat (was 1413 lines) | Removed dead ISR0 block, inlined single-use subroutines → 1044 lines | --- ## 📄 License This project is licensed under the MIT License — see the [LICENSE](LICENSE) file. --- ## 👤 Author **[Your Name]** - GitHub: [@your-username](https://github.com/your-username) - Course: [Your Course Name, e.g. Embedded Systems / Microcontrollers Lab] - Institution: [Your University/College Name]
